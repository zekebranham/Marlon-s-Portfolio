Windows Server Security Lab

Overview

This lab explores enterprise-level system configuration using Windows Server. The focus is on implementing real-world administration tasks, user access control, and security policies in a local testbed. These experiments replicate what IT administrators and security analysts handle in production environments.

Key Skills Demonstrated

  • Initial Server Configuration via SConfig
  • Remote Desktop and static IP setup
  • Active Directory role installation and configuration
  • Group policy application and ACL testing
  • Firewall rule creation and SMB permissions

Lab Environment

This project was executed using a local VirtualBox VM setup with the following components:

  • Windows Server 2022 (no desktop experience)
  • Oracle Linux VM (client system)
  • Remote access enabled for testing RDP security
  • SMB shares configured and mounted via Linux

Security Features Explored

  • Role-based access via Active Directory groups
  • Port-specific inbound/outbound firewall configuration
  • Audit policy configuration to detect unauthorized access
  • SMB protocol inspection and restriction
  • User login verification and restricted folder visibility

Reflection

Working directly with Windows Server elevated my understanding of domain-based access control, system hardening, and real-time threat prevention. This hands-on experience complements my scripting and programming background by grounding abstract concepts in real system administration tasks.
