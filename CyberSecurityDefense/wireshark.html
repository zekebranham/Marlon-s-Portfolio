Wireshark Protocol Analysis

Overview

My hands-on experience with Wireshark allowed me to conduct detailed protocol-level network inspections. These activities explored how data travels across networks and how protocol fields can be used to detect, trace, and diagnose network communication behaviors.

Protocols Studied

  • Ethernet: Analyzed MAC addressing and Frame Type interpretation, as well as ARP discovery mechanisms.
  • HTTP: Inspected headers, request types (GET, POST), and HTTP status codes to understand web communication flows.
  • IP: Investigated IP header fields such as TTL, source/destination addresses, and fragmentation handling.
  • TCP: Tracked the 3-way handshake, sequence/ack numbers, retransmissions, and round-trip time calculations.

Lab Highlights

  • Ethernet Packet Analysis: Ethernet Lab Report
  • HTTP Session Analysis: HTTP Lab Report
  • IP Packet Structure Study: IP Lab Report
  • TCP Flow & RTT Analysis: TCP Lab Report

Reflection

Analyzing traffic in Wireshark helped reinforce my understanding of how networks operate and how different protocol layers work together. From dissecting headers to monitoring live sessions, these labs were a foundational part of building network fluency.
