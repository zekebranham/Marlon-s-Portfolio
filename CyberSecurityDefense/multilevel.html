Multi-Level Security (MLS) System Simulation

Overview

This project simulates a Multi-Level Security system using both Python and C#. It models real-world clearance levels (Public, Limited, Secret, Top Secret) by enforcing access control on 50 files using user roles. A real Windows Server setup was also used to test this model at the system level.

Security Model

The system is based on the Bell-LaPadula security model, which restricts read access to data based on security clearance levels. Users can view files that are at or below their clearance level.

  • Level 1 – Public
  • Level 2 – Limited
  • Level 3 – Secret
  • Level 4 – Top Secret

Python Simulation

A Python script was created to:

  • Define users with credentials and security levels
  • Generate 50 files with randomized access levels
  • Authenticate users and display accessible files

This allowed for quick testing of access enforcement logic and user restrictions.

C# Console Application

A console-based C# version of the MLS system was developed to mirror the Python implementation while gaining experience with static typing and .NET. This version:

  • Utilizes lists of User and File objects
  • Handles login and role-based file filtering
  • Supports password-masked input and validation

Windows Server Demonstration

A full implementation of the MLS model was deployed to a configured Windows Server instance. NTFS file permissions and group policies were used to:

  • Create user roles and groups mapped to clearance levels
  • Assign file permissions using Access Control Lists (ACLs)
  • Verify real-time enforcement using test logins

Key Takeaways

This project demonstrates practical understanding of access control models and security enforcement mechanisms across programming and system-level implementations. It highlights how theoretical models can be tested and validated using simulation and live environments.
